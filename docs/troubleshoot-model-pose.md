# 트러블슈팅 — 모델이 잘못된 위치에 스폰됨

**증상:** 지정한 좌표와 다른 위치나 방향으로 모델이 스폰된다.

## 원인과 해결법

- **링크 원점 오프셋** — 메쉬 원점이 링크 원점과 일치하지 않는다. 메쉬를 수정하지 말고 `<visual>` / `<collision>`의 `<pose>`를 조정한다.
- **RPY 단위가 도(degree)** — SDF의 `<pose>`는 라디안을 사용한다.
- **상위 프레임 기준 혼동** — 자식 링크의 `<pose>`는 부모 기준이다. 월드 기준과 혼동하지 않도록 `relative_to` 속성을 명시한다.

<br>

## 체크리스트

1. SDF 원본과 스폰 커맨드의 좌표값을 대조.
2. 라디안 ↔ 도 변환 확인 (`90° = π/2 ≈ 1.5708`).
3. Gazebo GUI의 Entity Tree에서 실제 pose 확인.
