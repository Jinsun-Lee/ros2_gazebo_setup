# 좌표계 / 단위 / 원점 설정 규칙

CAD와 Gazebo 사이를 오갈 때 가장 흔한 버그의 원인이 좌표계와 단위의 불일치입니다.

## 단위

- **길이:** 미터 (m)
- **질량:** 킬로그램 (kg)
- **각도:** SDF에서는 라디안 (`<pose>`의 RPY는 라디안)
- **시간:** 초 (s)

Fusion 360에서 밀리미터로 작업했다면 둘 중 하나를 선택한다.
- 메쉬를 미터 단위로 내보내기, 또는
- SDF의 `<mesh>` 요소에 `<scale>0.001 0.001 0.001</scale>`을 적용.

<br>

## 좌표계 규칙 (ROS / Gazebo)

오른손 좌표계, 방향은 다음과 같다.

| 축   | 방향  |
|-----|------|
| +X  | 전방  |
| +Y  | 좌측  |
| +Z  | 위쪽  |

RPY(roll-pitch-yaw)는 고정축 X-Y-Z 순서를 따른다.

<br>

## 원점 배치

- **모델 원점**은 kinematic root 위치에 둔다 — 모바일 로봇은 보통 지면 접촉점, 정적 구조물은 장착면이다.
- **base_link** 링크 원점은 일반적으로 모델 원점과 일치시킨다.
- 자식 링크의 `<pose>`는 **부모 링크 기준**으로 표현한다 (SDF 1.7+는 `relative_to` 속성으로 명시 가능).

<br>

## Pose 포맷

```xml
<pose>x y z roll pitch yaw</pose>
```

모든 값은 SDF 기본 단위(미터 / 라디안)이다.

<br>

## 흔한 실수

- 메쉬 파일과 SDF 사이에서 mm와 m를 혼용.
- `<pose>`에서 라디안 대신 도(degree) 사용.
- 메쉬 원점과 링크 원점 불일치 — 메쉬를 수정하지 말고 `<visual>` / `<collision>`의 `<pose>`로 보정한다.
