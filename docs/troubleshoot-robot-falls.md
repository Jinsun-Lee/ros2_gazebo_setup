# 트러블슈팅 — 로봇이 바닥을 뚫고 떨어짐
**증상:** 스폰 직후 로봇이 중력으로 바닥을 관통하며 떨어진다.

<br>

## 원인과 해결법
- **`<collision>` 누락** — 로봇이나 바닥에 collision이 정의되지 않음.
- **collision 메쉬가 너무 작음** — visual과 크기가 맞지 않는다.
- **스텝 사이즈가 너무 큼** — `<max_step_size>`를 0.001 이하로 줄인다.
- **비현실적인 관성** — 질량이 0이거나 관성이 0이면 정의되지 않은 동작이 발생한다. 합리적인 값을 사용한다.

<br>

## 체크리스트
1. Gazebo GUI의 **View → Collisions**로 collision 형상 시각화.
2. SDF에서 `<mass>`와 `<inertia>` 값이 0이 아닌지 확인.
3. 물리 스텝 사이즈를 0.001로 낮춰본다.
4. 지면(ground_plane)이 월드에 포함되어 있는지 확인.
